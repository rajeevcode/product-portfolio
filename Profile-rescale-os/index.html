Rescale OS

An Operating System for the Creative Lifecycle and Ad Launch Workflow

Rescale OS is a TypeScript monorepo with a React 19 control surface, an Express and PostgreSQL API, and shared Zod contracts. It connects creative production, task automation, launch tracking, workspace security, German translation, and creative analysis in one workflow, built on a parent/child creative state machine with rule-driven automation, an ad launcher, and workspace-level row-level security.

“I built Rescale OS to bring fragmented creative and campaign operations into one structured, rule-governed system.”

The problem: creative operations were scaling faster than process discipline

As creative output grew, production tracking, naming, hand-offs, and ad launches became inconsistent. Different people followed different workflows, and the link between a script, its hooks and image variants, and the ad that finally went live was hard to see.

  • Manual creative lifecycle tracking with no shared source of truth
  • Loose parent-child relationships between scripts and their hook / image variants
  • Inconsistent creative-code naming, causing collisions and ambiguity
  • Weak visibility from script → hook/image → launched ad
  • Hand-offs (strategist → editor → launcher) coordinated manually
  • No automatic promotion of a parent when its children were launched or paused
  • Naming-compliance and QA gaps surfaced late, if at all
  • German translation of scripts/ad copy done ad-hoc per person
  • No durable, retriable automation when a status change should trigger downstream work
  • No workspace-level isolation guaranteeing one team couldn’t see another’s data

The core issue was lack of shared workflow visibility across creative production, review, and launch.

The solution: a rule-governed operating layer for creative execution

Rescale OS models creative work as a lifecycle, not a list of tasks, from concept and script, through rule-generated hook/image child tasks and editing, to launch readiness and the ad launcher.

Complete end-to-end application journey

The product flow, as verified by the e2e regression suite.

  1. Operator / admin signs in (workspace-scoped session)
  2. A script parent task is created (type VSL / UGC / REEL / STK)
  3. System generates a collision-proof creative code (e.g. VSL1-A)
  4. On Script Approved, rules auto-create hook (_HK) or image (_IMG) child tasks
  5. Child tasks move through editing and creative-review states
  6. Scripts / ad copy translated to German (Claude → Gemini → LibreTranslate)
  7. Creatives uploaded to Google Drive
  8. Child reaches READY_FOR_LAUNCH → an ad is auto-created at gandalf_approval
  9. Ad Launcher Hub tracks live status and ad category; launch is manual (FB API stubbed)
  10. When all children are LAUNCHED, the parent auto-promotes to LEARNING
  11. When all child ads are paused, the parent auto-promotes to DONE
  12. Ops Center surfaces QA, naming-compliance, and bottleneck events along the way
  13. Dashboards roll up sprint, product, strategist, and editor views
  14. Optimistic locking (revision tokens) keeps multi-user edits safe

Core modules

Core modules in the current Rescale OS build.

A

Creative Lifecycle & State Machine

Parent scripts and their hook/image child tasks move through a multi-state lifecycle with validated transitions and auto-promotion.

  • Parent types VSL / UGC / REEL → video hooks; STK → image ads
  • Status chain: Research → Idea/Script approval → Approved → Learning → Done
  • Parent auto-promotes SCRIPT_APPROVED → LEARNING when all children launched
  • Parent auto-promotes LEARNING → DONE when all child ads paused
  • Drag-and-drop fulfillment boards (Scripts / Videos / Images)
  • Optimistic locking via revision tokens (stale edit → 409)
B

Creative Code Generation

Collision-proof, database-sequenced creative codes that encode type, concept, iteration, and child relationship.

  • Script codes: {TYPE}{CONCEPT}-{ITERATION} (e.g. VSL1-A, STK2-C)
  • Excel-style iteration letters (A, B … Z, AA, AB …)
  • Child codes: {PARENT}_{HK|IMG}{N} (e.g. VSL1-A_HK1)
  • Sequence never resets, re-approval continues _HK4, _HK5 …
  • Per-workspace concept / iteration / child counters
C

Rule-Driven Task Automation

Status changes trigger downstream work through a durable, retriable automation queue.

  • Rules match on event, target status, and task type
  • Seeded rules auto-create video hooks (VSL) / image variants (STK) on Script Approved
  • Durable queue (task_automations): PENDING → PROCESSING → DONE / ERROR / DEAD_LETTER
  • Exponential backoff, up to 10 attempts, dead-letter on exhaustion
  • Ops worker drains the queue every 5 minutes
D

Ad Launcher Hub

Bridges ready creatives to ad launch, tracking live status and ad category. Launch is manual (Facebook API stubbed / mock).

  • Live status: gandalf_approval · new_test · live · paused · scheduled
  • Ad category: testing · potential · winning · super_winner · losing (manual)
  • READY_FOR_LAUNCH auto-creates the ad at gandalf_approval
  • Ad-account → campaign → ad-set selection; manual launch flow
E

Production & Winner Dashboards

Roll-up views for sprint and product health, and a winner-lifecycle workspace.

  • Production Dashboard: Overview / By Product / By Strategist / By Editor
  • Sprint health, velocity, revision leaderboard, NC-compliance chart
  • Winner Lifecycle: dashboard, pre-learnings, learnings library, capacity planning, fulfillment
  • Product scorecard (rank, live ads, hit rate, fulfillment, avg video length)
  • Charts via Recharts
F

Ops Center

Operations monitoring that surfaces QA and process events for action, the real “what needs attention” layer.

  • Event types: QA failure, bottleneck, revision pressure, NC gap, missing drive link, naming convention
  • Severity: critical / warning / info
  • Status: unresolved / resolved / dismissed / auto-fixed
  • Coaching text, how-to-fix, expected vs actual value
G

AI-Assisted Translation & Creative Analysis

AI supports German translation and creative analysis tasks that were previously manual.

  • German translation chain: cache → Claude (Sonnet) → Gemini (2.5 Flash) → LibreTranslate
  • Auto-fills translated metadata on task edit
  • Gemini creative analysis extracts 27 structured signals (CR_* fields) from video
  • Sync + async analysis jobs with pollable status
  • Filename-only fallback when file upload fails
H

Workspace Isolation (RLS)

Fail-closed, workspace-scoped data access enforced at the database.

  • RLS on creative_tasks, counters, rules, automations, launched_ads, idempotency keys
  • Fail-closed policy: access denied unless workspace matches or caller is admin
  • Session sets app.workspace_id / app.is_admin
  • Roles today: operator and admin
  • Workspaces: nptl and winner
I

API & Contracts Platform

A typed, shared contract layer keeps request/response shapes aligned across the stack.

  • Shared Zod schemas + TS types in packages/contracts
  • v2 task & ads APIs with HMAC-secured cursor pagination
  • Activity log per task (status-change history)
  • Helmet security headers; health checks with DB probe
  • Idempotency keys for safe retries

Technical breakdown

Data modeling

creative_tasks with parent/child links and per-workspace counters, launched_ads, creative_rules, task_automations, idempotency_keys, typed end to end via shared Zod contracts.

State machine

Multi-status creative lifecycle with validated transitions; parent auto-promotion to Learning (children launched) and Done (ads paused).

Automation queue

Durable task_automations queue: PENDING→PROCESSING→DONE/ERROR/DEAD_LETTER, exponential backoff, drained by a 5-minute ops worker.

State & concurrency

React Query + Zustand on the client; optimistic locking via revision tokens (stale edit → 409) on the server.

Security

Workspace RLS (fail-closed), admin/operator roles, Helmet headers, HMAC-secured pagination cursors.

AI integration

Provider fallback chain (Claude → Gemini → LibreTranslate) for translation; Gemini 27-field creative analysis with sync/async jobs.

Reliability & deploy

e2e Playwright regression suite, health checks with DB probe, Docker Compose + Kubernetes manifests.
