Problem

Creative and campaign operations were fragmented across multiple tools, dashboards, naming conventions, creative assets, and team workflows. Teams manually reviewed campaign setup, naming errors, performance signals, launch readiness, and winner/loser decisions, creating delays, inconsistent QA, and missed opportunities to scale winning creatives.

Solution

Built Creative Marketing Agents / Rescale Operator, a workflow-driven AI automation platform that combines campaign QA, creative signal filtering, winner detection, launch checklists, intelligence screens, dashboards, and human-in-the-loop review. One agentic workflow layer instead of disconnected spreadsheets and ad-hoc checks.

ClickUp API

Write · Idempotent
  • Task creation

The ClickUp Bridge is the only code path that creates tasks, idempotent hashes prevent duplicate tickets when daily checkpoint or diagnostic runs retry.

Technical breakdown

Architecture

Next.js (:3100) → FastAPI (:8001) → Redis/Arq worker → Supabase Postgres (snapshots, events, runs, operator log) + Ecom Profits read-only Meta view. Redis handles async jobs, SSE run events, and multi-worker agent task registry.

RAG & vector memory (Supermemory)

Vector search is not stored in Supabase, it runs on Supermemory, a managed semantic memory service. Supabase holds relational operator data only.

  • Ingest, supermemory_ingest.py scrubs secrets, attaches metadata (SOP, technical_doc, decision, metric_rule), idempotent custom_id → POST /v3/documents
  • Search, MemorySearchService builds query + filters (project: rescale_operator, optional module) → POST /v3/search → top-k chunks with similarity scores
  • RAG assembly, RagContextProvider returns context blocks, citations, confidence; degrades gracefully if Supermemory is disabled/timeout/error
  • Ask Operator, AgentOrchestrator merges live dashboard context + RAG prompt block → Hermes Agents / Claude → evidence + warnings (read-only, no writes)
  • Config gates, SUPERMEMORY_ENABLED, SUPERMEMORY_API_KEY, AGENT_RAG_ENABLED; Redis-backed task registry for multi-worker /api/agents/query
Data & persistence (Supabase)
  • Supabase Postgres via asyncpg, tenant-scoped reads, batch upserts, migrations (e.g. 0026_winner_notifier)
  • Tables: ad_launch_tracking, ad_winner_snapshots, ad_winner_events, operator runs, publish approvals
  • Ecom Profits read-only pool, default_transaction_read_only = on
  • Brand rules in config/brands.yaml
AI & agent layer
  • Hermes Agents, agent runtime provider for orchestrated tool use
  • OpenRouter, multi-model LLM routing layer
  • Claude provider, Ask Operator, Creative Agents, diagnostic Opus synthesis
  • Creative Agents, read-only image/video/research draft generation
  • Diagnostic engine, 4 diagnostics + Claude Opus synthesis
  • Winner notifier, rules classifier + Supabase event/snapshot schema
External API integrations
