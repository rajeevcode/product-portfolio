Creative Marketing Agents / Rescale Operator — shipped

Agent Memory & RAG

Read-only Ask Operator layer — Supermemory vector search for semantic retrieval, RagContextProvider for grounded prompt assembly, Hermes Agents + OpenRouter + Claude generation, and citation-backed evidence. Supabase stores operator runs; vectors live in Supermemory.

Architecture

1
Ingest → Supermemory vector store

SOPs, technical docs, decisions, metric rules → scrubbed → POST /v3/documents with metadata (project: rescale_operator).

↓
2
Query → semantic search

MemorySearchService → POST /v3/search → top-k chunks + similarity scores + citations.

↓
3
RAG context → orchestrator

RagContextProvider merges live dashboard context + retrieved blocks → Hermes Agents / Claude → evidence + confidence warnings.

Vector database

Supermemory (managed service) — not pgvector in Supabase. Embeddings and vector search are handled by Supermemory's API. Operator DB (Supabase Postgres) stores runs, snapshots, and approvals separately.

What shipped

supermemory_client.py, supermemory_ingest.py, supermemory_search.py, rag_context_provider.py, agent_orchestrator.py, /api/agents/query, Redis task registry, Agent Memory UI (NEXT_PUBLIC_AGENT_MEMORY).

Safety & degradation

Read-only Phase 1 — no Meta/DB writes. Supermemory outage returns empty RAG context + warning (never 500s). Low confidence (<0.4) flagged to user. Secret scrubbing on ingest.

Config gates

SUPERMEMORY_ENABLED, SUPERMEMORY_API_KEY, SUPERMEMORY_BASE_URL, AGENT_RAG_ENABLED, HERMES_AGENT_ENABLED, AGENT_WRITE_ACTIONS_ENABLED=false.
